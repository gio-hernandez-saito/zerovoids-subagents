## 평가 척도

풀이를 아래 6개 항목으로 평가합니다. ★ 5점 만점 (0.5 단위).

| 항목         | 설명                      | ★★★★★ 기준                 |
|------------|-------------------------|--------------------------|
| **정확성**    | 모든 케이스 통과 여부 (엣지케이스 포함) | 엣지케이스까지 빠짐없이 처리          |
| **시간복잡도**  | Big-O 기준 최적 여부          | 해당 문제의 알려진 최적해와 동일       |
| **공간복잡도**  | 추가 메모리 사용 최소화           | 불필요한 자료구조 없음             |
| **가독성**    | 변수명, 구조, 의도 전달력         | 코드만 읽고 풀이 전략이 바로 파악됨     |
| **언어 활용도** | JS/TS 내장 메서드·자료구조 적절 활용 | 언어 특성을 살린 간결한 구현         |
| **면접 설명력** | 풀이 근거를 구두로 설명하기 쉬운 정도   | 왜 이 접근인지, 트레이드오프까지 설명 가능 |

### 등급 기준

| 점수           | 의미              |
|--------------|-----------------|
| ★★★★★        | 최적. 개선할 게 없음    |
| ★★★★ ~ ★★★★½ | 좋음. 사소한 개선 여지   |
| ★★★ ~ ★★★½   | 보통. 동작하지만 개선 필요 |
| ★★ ~ ★★½     | 부족. 눈에 띄는 문제 있음 |
| ★ ~ ★½       | 심각. 근본적 재설계 필요  |

```

| 항목 | 점수 | 코멘트 |
|------|------|--------|
| 정확성 | ★★★★★ | ... |
| 시간복잡도 | ★★★★★ | ... |
| 공간복잡도 | ★★★★ | ... |
| 가독성 | ★★★★½ | ... |
| 언어 활용도 | ★★★½ | ... |
| 면접 설명력 | ★★★★ | ... |

**종합**: ★★★★¼ (6개 항목 평균)

**총평**: ...

**개선 포인트**: ...
```

## 개선 풀이 (`.best.js`) 작성 규칙

1. 원본 풀이와 **다른 접근**이 있을 때만 작성 (동일 로직의 미세 수정은 코멘트로 충분)
2. 파일 상단에 원본 대비 **무엇이 다른지** 주석으로 명시
3. 개선 근거를 평가 항목 기준으로 설명

```js
// 개선 풀이: 42888_오픈채팅방
// 원본 대비 변경점: ...
// 개선 항목: 시간복잡도 (O(n²) → O(n)), 가독성 (Map 활용)
```
