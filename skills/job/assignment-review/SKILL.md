> context: 프론트엔드 개발자. TypeScript 베이스. React, Vue 경험.

## 평가 척도

과제를 아래 6개 항목으로 평가합니다. ★ 5점 만점 (0.5 단위).

| 항목         | 설명                      | ★★★★★ 기준                      |
|------------|-------------------------|-------------------------------|
| **코드 품질**  | 일관성, 네이밍, 타입 안전성, 린트 준수 | 컨벤션 통일, 타입 완전, 불필요 코드 없음      |
| **아키텍처**   | 컴포넌트 분리, 관심사 분리, 확장성    | 구조만 보고 의도가 파악됨, 변경에 유연        |
| **기능 완성도** | 요구사항 충족, 엣지케이스 처리       | 명시 요구사항 + 암묵적 기대까지 충족         |
| **UX/UI**  | 사용성, 반응성, 에러 처리 UI      | 로딩/에러/빈 상태 처리, 반응형 대응         |
| **성능**     | 불필요한 리렌더, 네트워크 최적화      | 메모이제이션 적절, 번들 사이즈 의식          |
| **문서/커밋**  | README, 커밋 히스토리, 코드 주석  | README로 실행 가능, 커밋이 작업 흐름을 보여줌 |

### 등급 기준

| 점수           | 의미                    |
|--------------|-----------------------|
| ★★★★★        | 바로 PR 머지해도 될 수준       |
| ★★★★ ~ ★★★★½ | 좋음. 사소한 피드백만 있을 수준    |
| ★★★ ~ ★★★½   | 보통. 동작하지만 아쉬운 점 다수    |
| ★★ ~ ★★½     | 부족. 구조적 문제 있음         |
| ★ ~ ★½       | 심각. 요구사항 미충족 또는 실행 불가 |
