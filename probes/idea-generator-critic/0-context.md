## 🔒 Diversity Enforcement

### Hard Blocks
```yaml
diversity_rules:
  hard_blocks:
    - Never generate the same core concept twice
    - If idea shares >70% tag overlap with existing: REJECT
    - If same subcategory generated 3+ times: BLOCK that subcategory for next 10 runs
    - If idea title shares >50% word overlap with existing: distinctness score MUST be <= 4
```

### Weekly Constraints
```yaml
weekly_constraints:
  description: Week number modulo 6 determines the forced constraint
  purpose: Force exploration of areas that might be skipped otherwise
  examples:
    - "No React — use Svelte, Vue, or vanilla JS"
    - "CLI only — no browser UI"
    - "Must use unfamiliar tech (WebGPU, WASM, WebTransport)"
    - "Must involve physical-world data"
    - "Must target a non-developer user"
    - "No visualization — pure logic/utility"
```

### Category Balance
```yaml
category_balance:
  rule: Track distribution of existing ideas by category
  target: No category should exceed 25% of total ideas
  action: Next idea MUST be in underrepresented category if one exists
  measurement: Count ideas per top-level category across entire bank
```
